# FEAT-0011 — 점수판 · 킬 피드백 UI

| 항목 | 값 |
| --- | --- |
| 상태 | 완료 (2026-07-12), 점수판 표시 재검증 필요 |
| 관련 | [거점 점령전](./FEAT-0005-capture-point.md) 점수 코어 |

## 왜 넣는가 (의도)

점수 상황과 처치 피드백을 화면에서 바로 보이게 한다. 누가 몇 점인지(등수), 내가 왜 점수를 얻었는지(사유)를 즉시 전달.

## 동작 (코드 기준)

### 점수판 (우상단)
- 모든 플레이어를 **점수순 정렬** → **등수**(#1~#3 금·은·동), **프로필 얼굴**(HeadShot 썸네일, 캐시), 이름, 점수. 본인은 초록 강조.
- `leaderstats.Score` 기반, 0.5초마다 갱신.

### 킬 획득 팝업
- 처치로 점수를 얻으면 점수판 옆에 **`+N`(색) + 사유(흰 글씨)** 가 떠오르며 사라짐.
- 사유: `처치`(일반) / `거점 처치`(거점 내) / `N연속 처치`(연속킬). `+N`은 거점=노랑, 일반=초록.

### 킬 사운드 / 관중 환호
- **킬 사운드(Apex)**: 처치한 **본인만** 2D로 청취.
- **관중 환호**: 처치 시 **관중석(Seatbase) 8방향** 에미터에서 재생 → 맵 바깥에서 사방으로 들림(2초 디바운스).

## 신호 흐름

- 서버 `CapturePointSystem`: 처치 점수 확정 시 `ScoreGainEvent:FireClient(killer, gain, kInZone, streak)` + `playCrowd()`.
- 클라 `ScoreHUD`: 점수판 갱신 + 팝업 + 킬 사운드(본인 2D).

## 구현 위치

- `ServerScriptService.CapturePointSystem` — `ScoreGainEvent`(신규 RemoteEvent), 관중 환호 에미터.
- `StarterPlayer.StarterPlayerScripts.ScoreHUD` (신규) — 점수판·팝업·킬 사운드.

## 메모

- 크로스헤어는 **점(dot)만** 남기고 십자선 제거(별개 변경).
- ⚠️ 랭킹 점수판 렌더 표시는 Play 재검증 필요(알려진 이슈).

## 변경 로그
