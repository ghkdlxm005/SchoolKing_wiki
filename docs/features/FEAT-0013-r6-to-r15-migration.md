# FEAT-0013 — R6 → R15 전환

| 항목 | 값 |
| --- | --- |
| 상태 | 스크립트 대응 완료 · 애니메이션/오프셋 재작업 대기 |
| 관련 시스템 | 캐릭터 리그, 이동, 무기 3인칭, 카메라 |
| 날짜 | 2026-07-19 |

## 결정

- **아바타 타입 = R15 고정.** "플레이어 선택(Player Choice)"은 쓰지 않는다.
- **피격 판정은 기존 방식 유지** — R15로 세분화하지 않는다.

### 왜 R15 고정인가 (플레이어 선택을 피하는 이유)

"플레이어 선택"은 각 플레이어가 **자기 계정 설정대로** 들어온다. 한 서버에 R6와 R15가 섞일 수 있고, 그러면 리그를 가정한 코드·애니메이션·히트박스가 절반의 플레이어에게 어긋난다. 경쟁 FPS에서는 치명적이라, 리그를 하나로 고정하는 것이 정석이다.

## 리그 전환은 스크립트로 못 한다

R6/R15 전환은 **Game Settings > Avatar > Avatar Type** 설정이다. 커맨드 바 스크립트로 바꿀 수 없다. 이 설정은 사람이 직접 바꾸고, **그로 인해 깨지는 코드만** 아래에서 손봤다.

## 조사 결과 — 대부분 이미 R15 호환

리그 의존 가능 스크립트 31개를 전수 조사한 결과, 뼈대는 이미 양쪽을 염두에 두고 짜여 있었다.

| 시스템 | R15 상태 |
| --- | --- |
| **피격 판정** (전 무기) | ✅ `bodyMultiplier`가 R6·R15 부위 이름을 **모두** 보유 (`Left Arm` + `LeftUpperArm`/`LeftLowerArm`/`LeftHand` 등). 변경 없음 |
| 1인칭 뷰모델 | ✅ 카메라에 붙는 별도 리그. 캐릭터 리그와 무관 |
| 3인칭 무기 웰드 (`WeaponRig3P`) | ✅ 무기를 **HumanoidRootPart**에 웰드. R6·R15 파트 이름 화이트리스트 보유 |
| 카메라·아이템·스폰·봇 | ✅ 대부분 `HumanoidRootPart`(양쪽 존재) 폴백 보유 |

`RootPart`라는 이름이 31개에서 잡혔지만, `HumanoidRootPart`는 R6·R15 **양쪽에 모두 존재**하므로 대부분 그대로 동작한다.

## 스크립트 수정 (3곳)

| 파일 | 문제 | 조치 |
| --- | --- | --- |
| `SlideScript` | 이동 물리 기준이 `Torso` 우선(`Torso or HRP`). R15엔 Torso가 없다 | **`HRP` 우선**으로 뒤집음 (`HRP or Torso`) |
| `DummyCloseShooter` | 봇 조준·발사 지점이 `Torso` 전용 | `UpperTorso` / `HRP` 폴백 추가 |
| `LegCrutch.ServerHandler` | 3인칭 목발을 `"Right Arm"`(R6 전용)에 웰드 | **`RightHand`** 우선으로 전환 (R6 폴백 유지) |

`DeathShatter`도 `Torso`를 참조하지만 **HRP를 먼저** 찾으므로 R15에서 그 분기를 타지 않는다 — 무해, 변경 없음.

## 스크립트로 해결 못 하는 것 (Play 확인·재작업)

이 셋은 코드가 아니라 **콘텐츠·튜닝**이라 실기 확인이 필요하다.

1. **목발 3인칭 위치 오프셋** — `"Right Arm"` → `"RightHand"`로 붙게 고쳤으나, R6 팔과 R15 손은 위치가 달라 목발이 손에서 어긋나 보인다. `hubC0` / `MAIN_ARM_C0` 값을 R15 손 기준으로 **Play에서 재조정** 필요.
2. **커스텀 애니메이션** — 슬라이드·앉기·무기 3인칭 포즈가 전부 R6 제작이라 R15 리그에서 재생되지 않는다. 당장은 **기본 R15 애니**로 진행되고(스크립트는 에러 없이 넘어감), 추후 R15용 재제작 필요.
3. **카메라·앉기 높이** — `CameraHeadOffsetY = 1.5`, `CrouchCamOffset = -2.0`이 R6 비율 기준. R15는 키가 커서 미세하게 어긋날 수 있어 Play에서 튜닝.

## R6 vs R15 참고

| 항목 | R6 | R15 |
| --- | --- | --- |
| 파츠 수 | 6 | 15 |
| 캐릭터당 부하 | 가벼움 | 약간 무거움 (조인트·애니 계산 증가) |
| 히트 판정 | 굵직함 | 세밀함 (상박/하박/손 구분) |
| 스케일 | 통일 | 계정별 편차 (BodyScale) |

> **관중 NPC(800여 명)는 R15로 바꾸지 않는다.** 아바타 타입 설정은 **플레이어**에게만 적용되고, 수동 배치한 NPC는 그대로 R6로 남는다. 굳이 R15로 만들면 파츠·조인트가 2~3배가 되어 성능만 나빠진다.

> **스케일 고정 권장** — R15 플레이어는 계정마다 체격이 달라 뷰모델·카메라 정렬이 사람마다 미세하게 달라질 수 있다. 경쟁 공정성을 위해 `HumanoidDescription`으로 전원 체격을 고정하는 것을 권한다. (미적용)

## 구현 위치

- `StarterPlayer.StarterCharacterScripts.SlideScript` — RootPart HRP 우선
- `ServerScriptService.DummyCloseShooter` — R15 몸통 폴백
- `StarterPack.LegCrutch.ServerHandler` — RightHand 웰드

## 변경 로그

- 2026-07-19: R15 고정 결정. 스크립트 3곳 R15 대응. 애니메이션·오프셋 재작업은 대기.
