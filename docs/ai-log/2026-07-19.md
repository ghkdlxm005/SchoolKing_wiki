## 개발 항목

1. **랭킹 HUD** — 상위 4명을 등수·프로필 얼굴·점수로 표시
2. **HUD 화면 맞춤** — 화면 밖으로 넘치던 요소 수정 + 해상도 대응(모바일 지원)
3. **점수 획득 피드백** — 처치/거점 점령 사유를 영어로 표기, 획득 시점에 팝업
4. **거점 점수 지급 방식** — 1점씩 누적 → 1초마다 5점 일괄
5. **점수 표시 동기화** — 상단 SCORE와 랭킹의 갱신 타이밍 일치
6. **UI 미표시 버그 해결** — 새로 만든 HUD가 화면에 아예 안 나오던 문제

## 사람이 구조화해 전달 (Human)

- **배치 결정**: 랭킹은 화면 하단이 아니라 **상단 스코어 패널 바로 아래**, 실시간 획득 표시는 **점수 숫자 바로 옆**. (화면 스케치로 위치 지정)
- **규모 결정**: 동시 전투 인원이 최대 4명이므로 랭킹은 **4명까지만** 노출.
- **표기 규칙**: UI 텍스트는 **영어로 작성**한다 — 로블록스 자동 번역을 태우기 위함. 한국어 하드코딩 금지.
- **표시 방식**: 거점 점수는 상시 라벨이 아니라 **틱마다 팝업**으로. 점수도 화면 숫자와 같은 단위(5점)로 묶어서 오를 것.
- **일관성 요구**: 등수 색(금·은·동)을 랭킹 링·점수 텍스트·인게임 플레이어 외곽선에 **동일하게** 적용. → *이 중 외곽선 적용은 이후 철회됨(아래 참조).*
- **품질 기준**: "상단 SCORE와 랭킹 점수가 미세하게 어긋난다" → 타이밍을 **완전히 동일**하게 맞출 것.
- **모바일 전제**: 이 게임은 모바일도 지원하므로 UI가 **플레이어 화면에 맞춰** 나와야 한다.

## Claude 구현·디버깅 (AI)

- `ScoreHUD`(신규 LocalScript) — 등수 정렬, `GetUserThumbnailAsync` 프로필 캐시, 등수 색, 획득 팝업.
- `HUDResponsive`(신규 LocalScript) — 최상위 HUD 요소마다 `UIScale` 부착 + **위치 오프셋도 같은 비율로 축소**. 코너 앵커를 유지한 채 전체가 비례 축소되도록 함(단순 UIScale만으론 오프셋이 안 줄어 겹침이 남음).
- `CapturePointSystem` — 거점 점수를 `dt` 누적 방식에서 **1초 틱 정수 지급**으로 변경하고, 지급 시점에 `ScoreGainEvent(kind="zone")`를 클라로 발사해 **화면 숫자와 팝업이 같은 순간**에 뜨도록 함.
- **미표시 버그 추적** — GUI가 안 보이는데 스크립트는 정상 종료(`ready` 로그 출력). 순서대로 배제:
  1. 서버 데이터모델로 PlayerGui 조회 → 안 보임. 단 **클라가 만든 GUI는 서버에 복제되지 않으므로** 이 조회는 무효로 판단.
  2. `refresh()`를 `pcall`로 감싸 에러가 묵살되던 것을 warn 출력으로 바꾸고, 셀 개수·절대좌표·스케일을 콘솔에 찍음 → `cells=1`, 좌표 정상 = **그릴 준비는 끝난 상태**.
  3. 그리면 되는데 안 보인다 → **누가 끄고 있다**로 방향 전환. PlayerGui를 만지는 코드를 전수 검색해 `HUDController`가 `GameHUD` 외 모든 ScreenGui를 `Enabled=false`로 만들고 `ChildAdded`로 신규까지 끄는 것을 확인.
- 부수 수정 — `TopBell.Title`이 화면 위로 잘리던 좌표, `"👑 School King"` 대소문자 오타.

# 2차 — 이동 V1.0.1 · Monster Energy 아이템

## 개발 항목

1. **이동 V1.0.1 수치·동작** — 속도 재정의, 가속 2초, 관성·방향 전환·공중 제어
2. **리프트 패널티** — 리프트 탑승 시 표시/속도 불일치 해결
3. **Monster Energy 필드 아이템** — 획득 시 이동속도 증감
4. **리스폰 분산** — 다른 플레이어와 겹쳐 스폰되지 않도록
5. **전투 피드백** — 거점 점령 배너, 피격 시 빨간 글로우

## 사람이 구조화해 전달 (Human)

- **수치 확정**: 걷기 16 / 달리기 24 / 앉기 8 / 슬라이딩 50 / 점프 4. (초안의 슬라이딩 33·점프 6.8에서 플레이 체감으로 조정)
- **동작 스펙**: 가속 2초, 관성, 공중 제어, 방향 전환, 점프대·리프트 패널티.
- **아이템 설계**: Monster Energy를 필드 픽업으로. **+10 / 5초 → −3 / 2초 반동**, 소화기처럼 윤곽선, 위아래 호버링. **슬라이딩에는 +20**.
- **연출 요구**: 빨라진 것이 UI로 느껴져야 한다 → FOV로 속도감(옥테인 Q 참고), 값은 **+7 / −3**, 급격히 바뀌지 말고 서서히. 슬로우(빨강) 글로우는 폭 절반.
- **피격 피드백**: 데미지 입을 때도 아이템과 같은 방식의 **빨간 글로우**.
- **리스폰 규칙**: 다른 플레이어와 같은 곳에서 스폰되지 않도록, 3초 제한 정도.
- **버그 리포트**: "슬라이딩 후 리프트 타면 Walking인데 속도는 달리기", "슬로우가 안 먹는다", "**상태가 안 바뀌면 이속 적용이 안 된다**(이게 큰 듯)", "FOV가 중간에 갑자기 풀린다", "가끔 이속 증가가 안 먹는다".

## Claude 구현·디버깅 (AI)

- 이동 물리 레이어를 `SlideScript` 한 곳에 추가 — 관성·방향 전환·공중 제어는 결국 "휴머노이드가 입력에 속도를 즉시 스냅한다"는 **한 가지 원인**이라 감속·조향만 덮어쓰는 방식으로 통합. 가속은 기존 WalkSpeed 트윈이 담당하므로 건드리지 않아 서로 상쇄되지 않게 함.
- `MonsterEnergySystem`(서버) 신규 — Tool로 놓여 있던 4개를 필드 픽업으로 전환, 에러만 내던 기본 스크립트 제거, 호버링·회전·윤곽선·근접 획득·재생성.
- 속도 증감은 서버가 `SpeedBonus` 속성으로 알리고 `SlideScript`가 상태별 목표 속도에 합산하는 구조로 설계(서버 권위 유지, 클라는 반영만).
- **"상태가 안 바뀌면 적용 안 됨" 추적** — 속성 변경 핸들러를 292행에 넣었는데 `targetSpeedFor` 정의는 417행이라, 핸들러 안에서 그 함수가 `nil`이었다. 즉 값이 바뀔 때마다 핸들러가 조용히 죽고 `setState`가 불리는 순간(상태 전환)에만 반영됐다. **Lua 상위 스코프 캡처 시점** 문제.
- **"가끔 안 먹는다" 추적** — 속도를 `Config` 기본값으로 **직접 덮어쓰는 지점이 7곳**(슬라이드 종료·에러 복구·슬라이드 점프·맨틀·벽 점프·비정상 상태 복구·힐 종료) 남아 있어 보너스가 유실. 전부 `targetSpeedFor()` 경유로 교체.
- **FOV 튐 추적** — 무기 4종이 `TweenService`로 `camera.FieldOfView`를 트윈하는데 매 프레임 오프셋을 덧쓰고 있었다. 새 트윈이 **내 오프셋이 포함된 값에서 출발**해 값이 누적·이탈. 직접 쓰기를 포기하고 `FovBonus` 값만 알리고 **무기 쪽 트윈 목표값에 합산**하는 구조로 전환.

# 3차 — 보안: 툴박스 백도어 제거

## 개발 항목

플레이 중 가짜 **"Error 501"** 창이 떠서 커맨드 바 실행을 유도 → 원인 추적 및 제거.

## 사람이 구조화해 전달 (Human)

- 화면 스크린샷 제보. **커맨드 바에 붙여넣지 않음**(공격 성립 차단).
- 판단 요구: *"저거 지우면 확실히 문제 없나"*, *"복제되는 건 없나"*, *"관중은 써도 되나"* — 삭제 전 영향 범위를 먼저 확정할 것.
- **삭제 보류 지시** 후 출처부터 확인하도록 순서 통제.

## Claude 구현·디버깅 (AI)

- 소스 전문 검색으로 `Error 501` / `require(숫자)` → **0건**. 여기서 "없다"가 아니라 **"검색을 피하고 있다"** 로 방향 전환.
- 화면이 **블러 처리**된 점에 착안해 `BlurEffect` + `TextBox` + `ScreenGui` 조합으로 재검색 → 1건 적중.
- 다단계 로더 해독: 1단계 `Script`가 속성 `hm`의 **역순 문자열**을 뒤집어 하위 속성에 심는 구조 확인 → 같은 방식으로 뒤집어 **페이로드 원문 복원**(실행하지 않음).
- 자기복제 여부 검증: `Clone()` 호출 대상·`DescendantAdded`·`InsertService` 부재 확인 → **자가 전파 없음** 결론.
- 관중 NPC 스크립트 88개 전수 검사 후 **안전 판정** → 껍데기 2개만 분리 제거.

## 개발 전후 비교

| 항목 | 이전(Before) | 이후(After) |
| --- | --- | --- |
| **가짜 경고창** | `Workspace.Folder` 껍데기 Model에 501 로더 상주, 접속 시 표시 | 제거. 전역 재검사 **501 패턴 0건** |
| **외부 코드 통로** | `Script`인데 이름이 `PackageLink`인 것 **12개**가 속성에 숨긴 에셋 ID를 `require` | 12개 전부 제거, **PackageLink 0건** |

# 4차 — 스폰 방향 · 로그 정리 · 데미지 파이프라인 점검

## 개발 항목

1. **스폰 방향 버그 수정** — 맞춰둔 스폰 방향이 무시되고 반대를 보던 문제
2. **디버그 출력 정리** — 3인칭 리깅 작업 때 넣은 상태 로그 제거
3. **맵 중복 오브젝트 조사** — 콘솔 에러(x2)의 출처 확인
4. **총기 데미지 파이프라인 정리** — 현재 방식 문서화 + 검증 구멍 3건 발견

## 사람이 구조화해 전달 (Human)

- **원인 지목 요구**: "위치 다 맞춰 놨는데 왜 그럴까? 너니?" — 맵 데이터가 아니라 **코드 쪽 원인**을 먼저 의심하라는 지시.
- **조사와 조치의 분리**: "뭔지 알려주고, 삭제는 하지 말아봐" — 중복으로 **보이는** 것과 실제 문제를 구분해서 보고할 것. 판단은 사람이 한다.
- **임의 변경 금지**: "사운드 자체에서 내가 빼라고 한 게 아니면 빼지 마. 뭐가 같이 출력되는지 그런 것만 같이 보여주면 내가 말할게."
- **설명 수준 조정**: 전문 용어로 된 1차 설명에 "이해가 안 돼" → 동작 원리를 **일상어로 다시** 서술.

## Claude 구현·디버깅 (AI)

- **스폰 방향** — 두 배치 함수(`SpawnDistributor.placeCharacter`, `CapturePointSystem.scatterPlayers`)가 모두 `char:PivotTo(CFrame.new(위치))`를 쓰고 있었다. `CFrame.new(위치)`는 회전이 항등이라 SpawnLocation의 방향이 **전달 경로 자체가 없었다.** 수평 LookVector를 살려 `CFrame.lookAt`으로 교체. → [BUG-0009](../bugs/BUG-0009-spawn-orientation-reset.md)
- **디버그 출력** — `[LegCrutch] motion: ...` 로그를 3개 무기 핸들러에서 제거. **실리콘건 쪽은 복사 흔적으로 `[LegCrutch]`라고 잘못 찍히고 있었다** — 로그만 보면 목발 문제로 오인하기 쉬운 상태였다.
- **중복 오브젝트 조사** — 이름 중복 모델을 전수 수집한 뒤, **이름이 같은 것**과 **좌표까지 겹친 것**을 분리했다. 조명 12개·글로우스틱 92개·관중석 99석 등은 정상 배치. 실제 겹침은 3D 텍스트 간판 부근(약 250파트)과 글로우스틱 25곳. 콘솔 에러 `(x2)`의 원인인 `Hoop Fever`는 **6.4스터드 간격으로 나란히 놓인 아케이드 2대**로 중복이 아니었다. 에러 자체는 어트랙트 연출용 사운드 누락. 백도어 전례가 있어 해당 모델 스크립트 76개를 전수 검사(위험 패턴 0건). **조치 없음 — 의도된 배치로 확인.**
- **데미지 파이프라인** — 4정의 `ServerHandler`를 읽어 흐름을 정리하는 과정에서 입력값 검증 누락 3건 발견. → [BUG-0010](../bugs/BUG-0010-weapon-server-validation-gaps.md)

# 5차 — 사운드 개편

## 개발 항목

1. **피격·처치·사망 사운드 재정의** — 어디서 나고 누구에게 들리는지 명시
2. **관중 반응음** — 처치/사망 시 관중석에서 반응
3. **관중 앰비언스** — A/B 관중석이 번갈아 술렁이는 배경음
4. **라운드 종료 팡파레**

## 사람이 구조화해 전달 (Human)

- **두 축으로 명세**: 각 소리마다 **재생 위치**(내 몸 / 관중석 / 2D)와 **청취 대상**(나만 / 전원)을 분리해 지정.
- **교대 규칙을 직접 설계**: "A에서 먼저 나오고, `crowd_1`이 **50% 재생**되면 `crowd_2` 시작, `crowd_2`가 50% 되면 다시 `crowd_1`" — 겹침 비율까지 사람이 정함.
- **범위 제한**: "내가 빼라고 한 게 아니면 빼지 마. **뭐가 같이 출력되는지만 보여주면 내가 말할게.**"
- **되돌리기 지시**: `self_hit`은 내 몸 → **2D로 환원**. `Crowd Cheering`은 전원 재생 **유지**.

## Claude 구현·디버깅 (AI)

- **`ClientSFX` 모듈 신설** — "나한테만 들리게"를 위한 별도 필터는 불필요하다. **클라이언트가 생성한 `Sound`는 서버로 복제되지 않는다**는 성질을 그대로 쓰면 된다. 이 규칙을 `OnSelf` / `AtAudience` / `Play2D` 세 함수로 정리해 호출부에서 대상 제어를 신경 쓰지 않게 함.
- **사운드 조회 폴백** — `victory`가 `SFX.crowd` 하위에 있는데 `MatchFlowHUD`는 `SFX` 최상위만 조회하고 있었다. `SFX.<폴더>.<이름>` → `SFX.<이름>` → **재귀 탐색** 순 폴백을 넣어 호출부가 폴더 구조를 몰라도 되게 함.
- **`CrowdAmbience`** — 50% 지점 판정은 `TimePosition >= TimeLength × 0.5`. `TimeLength`는 로드 전 0이므로 최대 10초 대기 후 진행. 라운드 반복 시 루프 중첩을 막기 위해 **세대 카운터**로 이전 루프를 종료시킴.
- **범위 관리 실패와 정정** — 처치 시점이 겹친다는 이유로 기존 `playCrowd()`(전원 관중 환호)를 **요청 없이 주석 처리**했다. 사용자 지시로 복구. 이후 "임의로 건드리지 말고 동시 출력 목록만 보고하라"는 규칙이 확립됨.

## 개발 전후 비교

| 항목 | 이전(Before) | 이후(After) |
| --- | --- | --- |

# 6차 — 라운드 리셋 개편 · 관중 정비 · 사운드 전수 점검

## 개발 항목

1. **라운드 종료 리셋** — 위치만 옮기던 것을 부활과 동일한 상태 초기화로
2. **관중 정비** — 윤곽선이 뜨는 관중, 소리가 안 나는 관중석 해결
3. **뷰모델 잔존 버그** — 라운드 리셋 후 무기가 제자리에 남음
4. **컴퍼스 사운드** — 애니 마커 사운드가 한 번도 안 나던 문제 + 옛 경로 정리

## 사람이 구조화해 전달 (Human)

- **리셋 범위 지정**: "게임 자체가 끝난 판정이니, 쉴드·체력·탄창 등 **죽었다 다시 태어난 것처럼** 리셋해줘." — 개별 항목 나열이 아니라 **기준점(부활)** 을 제시.
- **증상 기반 지목**: "관중 쪽 어떤 건 소리가 나고 안 나고, 윤곽선이 어떤 건 있고 없고" — 원인 추정 없이 관찰만 전달. 조사와 판단은 분리.
- **범위 제한 재확인**: "사운드 자체에서 내가 빼라고 한 게 아니면 빼지 마. **뭐가 같이 출력되는지만 보여주면 내가 말할게.**"
- **맥락 제공**: 관중석 일부가 안 묶여 있는 이유를 "아직 작업자가 배치하고 안 묶어놔서" 라고 알려줌 → 임의 그룹 배정 대신 2D 재생으로 우회 결정.
- **경로 통일 지시**: "애니메이션에서 소리 나는 것만 하고", "옛날 경로는 없애줘."
- **범위 확정**: 나머지 무기 사운드는 "아직 구현 안 했으니 나중에 하나의 폴더로 정리해서" — 지금 손대지 않기로.

## Claude 구현·디버깅 (AI)

- **라운드 리셋** — `scatterPlayers`가 `PivotTo`로 위치만 옮기고 있었다. `p:LoadCharacter()`로 캐릭터를 새로 만들도록 바꾸니 체력·쉴드(`ShieldSystem.onChar`)·회복 아이템(`HealHandler.onChar`)·탄약(StarterPack 도구 재복제)·버프가 **기존 부활 경로를 그대로 타면서** 한 번에 초기화됐다. 개별 항목을 나열해 리셋하지 않은 이유다.
- **부수 충돌 차단** — 캐릭터를 새로 만들자 `SpawnDistributor`의 `CharacterAdded` 훅이 깨어나 A/B/C/D 배치를 덮어썼다. `Phase == "starting"` 동안 건너뛰도록 가드 추가.
- **관중 조사** — 관중이 **9개 구역**으로 흩어져 있고 이전 태그는 5개 구역(541명)에만 적용돼 **799명이 윤곽선 대상**이었다. 소리 쪽은 `FindFirstChild`가 동명 인스턴스 중 하나만 반환해 9곳 중 2곳에서만 났다. → [BUG-0014](../bugs/BUG-0014-audience-outline-and-ambience.md)
- **뷰모델 잔존** — 내가 만든 리셋 변경이 유입시킨 버그. `Died`는 안 뜨고(파괴이지 죽음이 아님), 정리 코드가 든 스크립트 자체가 Tool과 함께 삭제된다. → [BUG-0012](../bugs/BUG-0012-viewmodel-left-behind.md)
- **컴퍼스 사운드** — 콘솔 에러를 추적하다 **애니 마커 사운드 13개가 한 번도 재생된 적이 없다**는 걸 발견. `workspace:FindFirstChild("compass_sound")`가 nil이고 바로 아래 `if _sfx then`이 전체를 조용히 건너뛰고 있었다. → [BUG-0013](../bugs/BUG-0013-compass-marker-sound-path.md)
- **사운드 전수 점검** — 하나씩 콘솔에 뜨길 기다리는 대신 `PreloadAsync` + `TimeLength > 0`으로 전체를 한 번에 확인하는 방식을 만들었다. `PreloadAsync` 콜백의 상태값은 성공으로 보고했는데 실제로는 로드되지 않은 사례가 있어 **`TimeLength`를 기준으로 삼았다.**
- **작업 중 실수** — 주석 처리 루프가 자기가 만든 주석을 다시 찾아 **중첩 주석 7겹**을 만들었다. 이후 해당 줄을 통째로 교체해 정리.

# 7차 — R6 → R15 전환 · 관중 함성 위치 변경

## 개발 항목

1. **R6 → R15 전환** — 게임 리그를 R15로 고정, 스크립트 대응
2. **관중 배경 함성** — `audience A/B` → `audience Back`에서 3D 재생

## 사람이 구조화해 전달 (Human)

- **범위 명확화**: "R6 → R15 했을 때의 문제점?"을 먼저 물어, **판단 근거를 확보한 뒤** 결정. "플레이어 선택" 설정의 위험(리그 혼재)을 확인하고 **R15 고정**으로 확정.
- **불변 조건 지정**: "피격 판정은 동일하게 진행" — R15로 세분화하지 말 것.
- **역할 분담**: "리그 설정은 내가 바꿀게" — 설정 변경(Game Settings)은 사람이, **스크립트 대응은 AI**가.
- **애니 방침**: "기본 R15로 진행하고 스크립트 부분에서 바꿔줬으면" — 커스텀 애니 재작업은 나중, 지금은 코드만.
- **후속 지시**: 관중 함성을 `audience Back`에서, 패턴 동일(50% 교대), 3D로.

## Claude 구현·디버깅 (AI)

- **리그 의존 전수 조사** — R6 전용 파트(`Torso`)·사지 이름·조인트 이름·구형 BodyMover·Motor6D를 전 스크립트에서 검색. R6 조인트 이름(`Left Shoulder` 등)은 전부 **관중 NPC의 기본 Animate 스크립트**였고, 이들은 게임 아바타 설정과 무관하게 R6로 남으므로 제외.
- **피격 판정 검증** — 4개 무기 `bodyMultiplier`가 이미 R6·R15 부위 이름을 **모두** 갖고 있음을 확인. "피격 판정 동일 유지" 요구에 부합 → 변경 없음.
- **핵심 R6 의존 3곳 수정** — `SlideScript`(이동 루트 HRP 우선), `DummyCloseShooter`(봇 몸통 폴백), `LegCrutch.ServerHandler`(3인칭 목발 웰드 `Right Arm` → `RightHand`).
- **3인칭 웰드 구조 확인** — `WeaponRig3P`가 무기를 **HRP에 웰드**하고 파트 이름 화이트리스트에 R6·R15를 모두 보유 → 목발을 제외한 무기는 자동 호환.
- **한계 명시** — 목발 오프셋(팔↔손 위치 차)·커스텀 애니(R6 제작)·카메라 높이(R6 비율)는 스크립트로 해결 불가. Play 재작업 항목으로 분리 보고.
- **관중 함성 이동** — `CrowdAmbience`의 방출 대상을 `audience A/B`에서 `audience Back`으로 변경. 교대 패턴(50%)·3D·전수 수집은 유지.

## 남은 과제

- 실리콘건 장착 중 좌클릭 미발사 버그
- 1인칭 뷰모델 벽 클리핑
- 루팅 시스템 개편(책 드롭·자동 흡수) — **쉴드템 줍기는 아직 `Touched`(0.9 구슬 접촉) 방식**이라 Monster Energy처럼 거리 판정으로 통일 필요
- 이동 물리 체감 튜닝(`InertiaDecel` / `TurnDecel` / `AirControl`)
- 툴박스 에셋 추가 시 `SecurityScan` 습관화
- **무기 발사 입력값 검증 3건** — `origin` 미검증 / `direction` 각도 미검증 / **토스터 펠릿 개수 무제한(치명)**. → [BUG-0010](../bugs/BUG-0010-weapon-server-validation-gaps.md)
- **타격음 2종 재생 불가** — `shield_hit` / `hp_hit` 에셋 보관 해제 또는 대체 필요. → [BUG-0011](../bugs/BUG-0011-archived-hit-sound-assets.md)
- **처치 시 관중음 4겹** — 볼륨/구성 조정 여부 청취 후 결정
- **앉기 카메라 측정 실패** — 진단 로그상 `measured=-0.07`로 실측이 사실상 실패, 고정값 `-2.00` 폴백으로 동작 중. 측정 로직 보완 또는 진단 로그 제거 필요
- **무기 사운드 통합** — 컴퍼스만 애니 마커 방식으로 정리됨. 나머지는 미구현이며 추후 `SFX/<무기명>` 한 폴더로 모으기로 함. 목발·토스터·실리콘건은 현재 발사음·재장전음을 **공유**해 구분되지 않고, `GunSystem.Sounds.LegCrutch.Melee`는 ID가 비어 있음
- **컴퍼스 3D 사운드 부재** — 옛 경로 제거로 다른 플레이어에게는 컴퍼스 소리가 들리지 않음. 필요 시 마커 재생을 서버 브로드캐스트로 확장
- **라운드 시작 무적** — 리스폰 무적 5초가 캐릭터 생성 시점부터라 GAME START 후 1~2초 남음
- **R15 목발 오프셋 재조정** — `RightHand` 기준 `hubC0`/`MAIN_ARM_C0` 값을 Play에서 맞춰야 함
- **R15 커스텀 애니 재제작** — 슬라이드·앉기·무기 3인칭 포즈 (R6 제작본은 R15에서 미재생)
- **R15 카메라/앉기 높이 튜닝** — `CameraHeadOffsetY`·`CrouchCamOffset`이 R6 비율 기준
- **R15 스케일 고정 검토** — `HumanoidDescription`으로 전원 체격 통일(경쟁 공정성)
