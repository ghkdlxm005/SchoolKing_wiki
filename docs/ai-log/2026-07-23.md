## 개발 항목

1. **R15 슬라이딩 벽 통과** — R15 전환 후 슬라이딩이 벽을 관통
2. **관중 처치/사망 사운드 재배치** — audience Back → A·B, 대상 세분화
3. **발소리 튜닝** — 달리기 발소리 정상화, 상대 달리기 템포 조정
4. **힐(쉴드) 취소 조건 축소** — 앉기로 끊기던 것 제거
5. **콘솔 정리** — Hoop Fever 어트랙트 에러 제거

## 사람이 구조화해 전달 (Human)

- **증상 기반 지목**: "R15로 바꾸고 슬라이딩하니 벽 통과" / "슬라이딩 해결 안 됐어" — 관찰만 전달, 원인 추정은 AI가.
- **사운드 대상 명세**: "사망한 건 본인한테만 3D, 처치는 전체 3D" / "Back에서는 kill·death 안 나고, 관중 A·B에서 재생." — 재생 위치와 청취 대상을 분리해 지정.
- **발소리 판단 위임 후 정정**: 본인/상대 발소리가 다르게 들리는 문제를 보고받고, "출력 방식이 다르다, 내 발소리는 2D가 맞다, 상대 달리기 템포만 1.3배" 로 방향 확정.
- **범위 축소 규칙**: "쉴드 중 끊는 스킬은 달리기뿐. 앉기 눌러도 안 끊기게."
- **의도된 배치 존중**: Hoop Fever 관련 반복 에러는 모델을 건드리지 말고 **스크립트만 비활성화**.

## Claude 구현·디버깅 (AI)

- **슬라이딩 벽 통과** — 세 겹의 원인을 순차 확인했다. ①슬라이드가 몸통 충돌까지 끔(R15는 몸통이 벽 충돌 담당) ②지면 스냅이 물리 루트(R15는 HRP)를 **매 프레임 텔레포트** ③`LinearVelocity` MaxForce 100000이 벽을 밀고 나감. 셋 다 R6→R15 전환 부작용. → [BUG-0015](../bugs/BUG-0015-r15-slide-wall-clip.md)
- **처음 시도의 한계 인정** — 첫 수정(몸통 충돌 유지)만으로 안 잡혔고, 사용자의 "아직 통과한다" 피드백을 받고 텔레포트·고force 두 원인을 마저 찾았다.
- **관중 사운드 재배치** — `ClientSFX.AtAudienceAB` 추가(클라, 본인만 3D). 사망 `death_crowd`는 클라에서 A·B 본인만, 처치 `kill_crowd`는 서버가 A·B에서 전원 3D. Back의 kill/death 재생은 제거.
- **발소리** — 오해로 본인/상대를 3D로 통합했다가, 사용자 정정("내 발소리는 2D") 후 원복. 결과: 본인 2D(0.3), 상대 3D(1.5), **상대 달리기만 PlaybackSpeed 1.3**. 별개로 `IsRunState` 플래그를 추가해 달리기 가속 구간에 걷기 소리가 섞이던 것도 수정.
- **힐 취소** — 취소 트리거가 `LeftShift`(달리기) + `LeftControl`(앉기) 둘이었다. `LeftControl` 제거 → 달리기만 취소.
- **콘솔 정리** — Hoop Fever 2대의 `Attract`/`AttractMusic` 스크립트 4개 `Disabled`. 모델·배치는 무손상.
