# UI / HUD 시스템

## 책임

체력/쉴드 바, 탄약·게이지, 데미지 표기, 점수판, 적 외곽선 등 화면 표시 전반.

## 표시 항목

| 요소 | 소스 시스템 / 스크립트 |
| --- | --- |
| 체력·쉴드 | [전투/데미지](../combat/combat-damage-system.md) |
| 탄약·게이지 | [탄약/게이지](../combat/ammo-system.md) |
| 점수판·킬 피드백 | [점수/랭크](../meta/scoring-rank-system.md), `ScoreHUD` |
| 데미지 숫자·히트마커 | `DamageNumbers` (공유) |
| 상대 머리 위 쉴드 바 | `OverheadShield` |
| 적 외곽선 | `EnemyOutline` |

## 크로스헤어 / 조준

- **스카우터 스코프 제거** — 화면 중앙 크로스헤어 상시 표시. 우클릭은 FOV 줌만(스코프 비주얼 없음).
- **현재: 점(dot)만 표시** — 십자선(Top/Bottom/Left/Right)은 숨김, 중앙 `CenterDot`만 사용. (2026-07-12)
- `CrosshairController`의 탄퍼짐 연동 벌어짐 로직은 십자선용이라 화면상 효과 없음(코드는 유지).
- 힙파이어 정확도 배경 → [FEAT-0006](../../../features/FEAT-0006-hipfire-accuracy.md)

## 데미지 표기 (2026-07-12 개편)

- **단일 데미지**: 명중 지점 **좌측**, 부위별 색 — 머리 **노랑** / 몸통 **흰색** / 팔다리 **회색**. 동시 명중이 겹치지 않도록 좌우 지터.
- **합산**: **우측**에 같은 대상 0.7초 내 누적 합계 표시(점사·샷건·장판 틱이 하나로 합쳐짐).
- **투사체**(캔 폭발·컵 파편)도 숫자 표시, `AlwaysOnTop`이라 **벽 너머로도 수치 확인** 가능.
- 무기별 중복 표시 문제 → [BUG-0006](../../../bugs/BUG-0006-damage-number-duplicate.md)

## 랭킹 / 점수 획득 표시

- **랭킹 바**: 상단 스코어 패널 바로 아래에 **상위 4명**(동시 전투 인원)을 `#1~#4` 등수·**프로필 얼굴**·점수로 표시. 본인은 프로필 링에 흰 테두리.
- **등수 색**: 랭킹 링·점수 텍스트에 적용. **인게임 플레이어 외곽선에는 적용하지 않는다** — 등수 색을 입혔더니 해당 외곽선이 연막 가림 판정을 무시하고 계속 보이는 문제가 있어 철회했다. 외곽선은 흰색 단일.
- **획득 팝업**: 점수를 얻는 순간 스코어 숫자 옆에서 `+N + 사유`가 떠오른다 — `Kill` / `Zone Kill` / `Kill Streak xN` / `Zone Hold`.
- **표기는 영어**로 통일한다(로블록스 자동 번역 활용). UI 문자열에 한국어를 하드코딩하지 않는다.
- 랭킹은 상단 SCORE와 **동일한 `leaderstats.Score` 변경 이벤트**로 갱신되어 표시 시점이 어긋나지 않는다.

→ [FEAT-0011](../../../features/FEAT-0011-score-hud-kill-feedback.md), [점수/랭크](../meta/scoring-rank-system.md)

## 화면 크기 대응 (모바일)

이 게임은 모바일을 지원하므로 HUD는 **플레이어 화면에 맞춰 비례 축소**된다. `HUDResponsive`가 최상위 HUD 요소마다 `UIScale`을 부착하고 **위치 오프셋도 같은 비율로 축소**해, 코너 배치를 유지한 채 좁은 화면에서도 화면 밖으로 나가지 않는다(기준 1280×720, 0.42~1.0). 크로스헤어는 조준 기준이라 축소에서 제외한다.

:::warning 새 HUD를 추가할 때
`GameHUD.HUDController`는 허용 목록(`KEEP_GUI`)에 없는 ScreenGui를 **비활성화**한다. 새 UI를 만들면 이 목록에 이름을 등록해야 화면에 나온다. → [BUG-0007](../../../bugs/BUG-0007-hud-screengui-whitelist.md)
:::

## 적 외곽선

상시 **흰색** 외곽선(`Occluded`, 투명도 0.5) — 등수별 색상은 적용하지 않는다.

**적이 연막 안에 있으면 숨김.** 단 나도 같은 연막 안에 있으면 보인다. 연막에 가려질 때는 이름표·체력·머리 위 실드바도 함께 숨는다(`SmokeHidden`).

> 한때 **나→적 시선 선분이 연막을 통과하면 숨김**(둘 다 연막 밖이어도 가림)으로 확장했으나, 원래 동작으로 되돌렸다.
→ [FEAT-0009](../../../features/FEAT-0009-fire-extinguisher-smoke.md)

## 크로스헤어 장탄 UI (기획 · `26.06.21`)

전투 중 **남은 장탄을 크로스헤어 주변에서** 직관적으로 보이게 한다(화면 구석 숫자 대신 주변시야로 파악). 조준 방해 안 되게 크로스헤어에서 **살짝 떨어진 위치**.

무기별 표시 예시:

| 무기 | 표시 |
| --- | --- |
| 토스터(더블배럴) | `■■` 2발 / `■□` 1발 / `□□` 장전 필요 |
| 실링폼/실리콘건 | 폼 게이지 `██████` → `███□□□` → `█□□□□□` |
| 컴퍼스 / 목발 | 작은 칸 여러 개 + 숫자 보조 |

> 상태: 기획. 구현 시 기존 [탄약/게이지](../combat/ammo-system.md) 어트리뷰트(`Ammo`/`AmmoReserve`)를 크로스헤어 UI에 바인딩.
