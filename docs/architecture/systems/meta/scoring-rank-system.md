# 점수 / 랭크 시스템

## 책임

거점 점령전 점수 집계, 승리 판정, 등수 표시, 처치 피드백.

## 현재 구현 (코드 기준 · 2026-07-12)

점수는 플레이어별 `leaderstats.Score`(IntValue)에 누적된다. 관리 주체는 `ServerScriptService.CapturePointSystem`.

### 점수 획득

| 상황 | 점수 |
| --- | --- |
| 일반 처치 | **20** |
| 거점 안에서 처치 | **50** |
| 연속 처치 (거점 내) | 2연속부터 **+30씩** 누적 |
| 거점 유지 | **5점/초** (거점 안에 있는 **모든** 플레이어가 각자 획득) |

- **연속 처치는 거점 밖으로 나가면 리셋**된다.
- 자살(자기 캐릭터)은 점수에서 제외, 마지막 피격 후 10초가 지난 사망은 처치로 보지 않는다.

### 승리 조건

- **1000점 선착** 또는 **제한시간 5분(300초) 종료 시 최고점**.
- 종료 후 `RESET_DELAY`(6초) 뒤 라운드 리셋.

### 처치 피드백

- 서버가 처치 점수 확정 시 `ScoreGainEvent:FireClient(killer, gain, kInZone, streak)`.
- 클라이언트(`ScoreHUD`)가 **`+N` + 사유**(처치 / 거점 처치 / N연속 처치)를 팝업.
- **킬 사운드**(Apex)는 처치한 **본인만** 2D로 청취.
- **관중 환호**는 관중석 8방향 에미터에서 재생(맵 바깥).

### 등수 표시

- 우상단 점수판: 점수 내림차순 정렬 → **등수(#1~#3 금·은·동) · 프로필 얼굴 · 이름 · 점수**, 본인 강조.
- 상세 → [FEAT-0011](../../../features/FEAT-0011-score-hud-kill-feedback.md)
