**버그 내역**

**제목: 컴퍼스 리로드 애니메이션 동작 문제 — 원인 + 해결**

안녕하세요! 컴퍼스 리로드 애니에서 cylinder/capRoot가 안 움직이던 문제 해결됐어요. 다음 작업에 참고하시라고 정리합니다.

### 문제였던 것

- 애니메이션 자체는 정상이었음 (cylinder 회전 39° 등 키프레임 데이터 잘 들어가있었음)
- 그런데 게임에서 cylinder, capRoot 같은 부품들이 시각적으로 안 움직임
- Main 본체랑 팔은 잘 동작했는데 **그 안의 sub-parts만 정지**

### 원인

컴퍼스 viewmodel rig 안에 **한 파트에 두 가지 joint가 동시에 묶여있는 상황**이 있었음.

예를 들어:

- coal_1 ← capRoot에 Weld로 묶임 (rigging 의도)
- coal_1 ← HRP에 WeldConstraint로 또 묶임 (코드가 자동 추가)

이러면 엔진이 **"두 anchor를 동시에 만족시키는 건 불가능하니까 애니메이션 무시"** 해버림.

### 해결한 방식

무기 장착 코드(setupViewmodel)에서 자동 weld 추가하기 전에 **이미 Weld로 묶인 파트는 제외**하도록 수정. 이제 중복 묶임이 발생하지 않아서 nested 본도 정상 동작.

### 앞으로 rigging 시 주의사항

1. **한 파트는 한 joint chain에만 속하게** — 한 부품이 capRoot에도 묶이고 HRP에도 따로 묶이는 식의 이중 연결 X
2. **Motor6D + Weld 조합은 OK** — 메인 본은 Motor6D, 같이 움직일 sub-parts는 Weld로 묶는 방식 자유롭게 가능
3. **rig 변경 시 애니메이션도 같이 재작업 필요** — 본 hierarchy(부모-자식 관계)가 바뀌면 옛 애니가 새 rig에서 깨짐

### 확인 방법

새 rig 작업 후 게임에서 애니 재생했을 때 **특정 부품만 안 움직이면** 위와 같은 이중 joint 문제 의심. 알려주시면 같은 방식으로 fix 가능.
