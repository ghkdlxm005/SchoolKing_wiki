# 문서 수정하기

깃(Git)을 전혀 몰라도, 웹 브라우저만으로 문서를 고칠 수 있습니다.
처음 한 번만 **GitHub 계정**으로 로그인하면 됩니다 (무료 가입).

## 한눈에 보는 순서

1. 고치고 싶은 문서를 위키에서 엽니다.
2. 문서 맨 아래 **"Edit this page"** 를 누릅니다.
3. GitHub 편집 화면이 열립니다. 글을 고칩니다.
4. 초록색 **Commit changes...** 버튼을 누릅니다.
5. 1~2분 뒤 위키에 자동 반영됩니다. 끝!

## 자세히 따라하기

### 1단계 — 수정 버튼 찾기

문서 맨 아래로 내리면 연필 아이콘과 함께 **Edit this page** 글자가 있습니다. 이걸 누르세요.

### 2단계 — GitHub 로그인 (처음 한 번만)

GitHub 계정이 없다면 https://github.com 에서 무료로 가입하세요.
가입 후, 팀 운영자(민지)에게 **GitHub 아이디를 알려주면** 편집 권한을 추가해 줍니다.

### 3단계 — 글 고치기

편집 화면은 메모장처럼 생겼습니다. 글자를 그냥 고치면 됩니다.
서식(굵게, 제목, 목록 등) 쓰는 법은 [4. 노션처럼 꾸미기](../04-%EC%84%9C%EC%8B%9D-%EC%93%B0%EA%B8%B0/index.md) 를 참고하세요.

### 4단계 — 저장하기

화면 오른쪽 위 또는 아래의 초록색 **Commit changes...** 버튼을 누릅니다.
설명을 적는 칸이 나오면 무엇을 고쳤는지 짧게 적고(예: `리스폰 시간 수정`) 다시 **Commit changes** 를 누르면 끝입니다.

:::tip 실수해도 괜찮아요
모든 수정 기록이 남기 때문에, 잘못 고쳐도 언제든 되돌릴 수 있습니다. 마음 편히 고치세요.
:::

## 새 문서 만들기

1. 위키 저장소 https://github.com/ghkdlxm005/SchoolKing_wiki 로 갑니다.
2. `docs` 폴더로 들어갑니다.
3. 오른쪽 위 **Add file → Create new file** 을 누릅니다.
4. 파일 이름을 `폴더이름/문서제목.md` 형식으로 적습니다. 예: `기획서/26 06 07.md`
5. 내용을 적고 **Commit changes** 를 누르면 새 문서가 위키에 생깁니다.

:::note 파일 이름은 꼭 `.md` 로
문서 파일은 반드시 `.md` 로 끝나야 위키에 글로 보입니다.
:::
