# 운영 · 백업 가이드 (운영자용)

> 이 문서는 위키에 올라가지 않는 운영자(민지)용 메모입니다.

## 1. 평소 작업 흐름

PC에서 문서를 고친 뒤에는 **`push.bat` 더블클릭** 한 번이면 커밋 + 업로드 + 사이트 반영이 자동으로 됩니다.
친구들은 웹 편집기(`/admin`)에서 직접 수정하므로 PC 작업이 필요 없습니다 (사용법은 위키의 "위키 사용법" 참고).

## 2. 정기 백업

저장소 자체가 GitHub에 있어 1차 백업은 자동입니다. 추가로 월 1회 정도 전체 백업을 권장합니다.

### 방법 A — 압축 파일로 받기 (가장 간단)

1. https://github.com/ghkdlxm005/SchoolKing_wiki 접속
2. 초록색 **Code** 버튼 → **Download ZIP**
3. 받은 ZIP을 외장하드나 클라우드(구글 드라이브 등)에 날짜를 붙여 보관
   예: `SchoolKing_wiki_백업_2026-06.zip`

### 방법 B — 명령어로 받기

```bash
git clone https://github.com/ghkdlxm005/SchoolKing_wiki.git 백업폴더이름
```

### 노션 원본 백업

최초 노션 export 원본(`notion-export-backup.zip`)은 저장소 바깥에 따로 보관되어 있습니다.
이 파일도 클라우드에 한 부 올려두면 안전합니다.

## 3. 용량 주의점

GitHub 무료 저장소는 권장 용량이 약 1GB, 파일 1개 최대 100MB입니다. 현재는 여유가 많습니다.

- 이미지는 한 장당 2~3MB 이하로 올리도록 안내하세요.
- **대용량 3D 파일**(`.fbx`, `.obj`, `.rbxm`, `.rbxanim`)은 위키 표시용이 아니므로, 많아지면 별도 클라우드(구글 드라이브)로 옮기고 위키에는 링크만 두는 것을 권장합니다.
- 현재 저장소 용량은 약 120MB 수준입니다.

## 4. 문제가 생겼을 때

- **사이트가 안 떠요** → https://github.com/ghkdlxm005/SchoolKing_wiki/actions 에서 가장 최근 실행이 빨간 X 인지 확인. 빨간 X를 누르면 원인이 나옵니다.
- **빌드 실패** → 보통 문서 문법 오류입니다. 마지막으로 수정한 문서를 의심하세요.
- **되돌리고 싶어요** → 저장소의 모든 변경은 기록이 남습니다. 특정 문서의 "History" 에서 이전 버전으로 복구할 수 있습니다.

## 5. 기술 메모

- 사이트 엔진: Docusaurus 3.10 (정적 사이트)
- 배포: GitHub Actions → GitHub Pages (main 브랜치 push 시 자동)
- 검색: `@easyops-cn/docusaurus-search-local` (오프라인, 한국어)
- 다이어그램: `@docusaurus/theme-mermaid`
- 웹 편집기: Decap CMS (`static/admin/`) + Cloudflare Worker OAuth (`cms-worker/`)
- 문서 구조: 모든 페이지는 `폴더/index.md` 형태 (CMS 트리 편집을 위해 통일됨)
- 설정 파일(`docusaurus.config.js`)의 **주석은 반드시 영어로** 작성하세요. 한글 주석이 빌드 오류를 일으킨 적이 있습니다.
